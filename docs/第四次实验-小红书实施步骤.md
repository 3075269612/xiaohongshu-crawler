# 第四次实验：小红书方向实施步骤

## 当前选择

目标平台：小红书。

采集策略：人工登录、低频采集、遇到验证码或异常访问立即暂停，不绕过真实平台风控。

## 第一步：准备起始用户队列

将你计划采集的用户主页 URL 写入：

```text
social_media_crawler/data/xhs_user_queue.txt
```

每行一个，例如：

```text
https://www.xiaohongshu.com/user/profile/用户ID
```

也可以先改配置文件：

```text
social_media_crawler/config/xiaohongshu.example.json
```

## 第二步：先验证解析器

```powershell
uv run pytest social_media_crawler/tests/test_xhs_parser.py
```

如果暂时没有 `pytest`，可以先安装：

```powershell
uv add pytest
```

## 第三步：小规模浏览器试采

需要安装 DrissionPage：

```powershell
uv add DrissionPage
```

启动：

```powershell
uv run python -m social_media_crawler.src.xhs_manual_client --browser --config social_media_crawler/config/xiaohongshu.example.json
```

程序会打开浏览器。请手动登录小红书，遇到验证码时人工完成。程序会低频滚动、提取笔记链接、进入详情页解析标题、正文、标签、图片、视频等字段。

## 第四步：查看输出

```text
social_media_crawler/data/xhs_posts.jsonl
social_media_crawler/data/xhs_statistics.json
```

## 第五步：扩展到最终目标

最终目标是：

- 100+ 用户
- 5000 条帖子
- 50+ 视频帖

建议先跑 3-5 个用户验证字段质量，再逐步扩大队列。
